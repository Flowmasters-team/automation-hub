# Кейс: AI-бот для сервисной компании

## Проблема

Сервисная компания (ремонт, установка, обслуживание) теряла до 40% входящих заявок из Авито и мессенджеров. Менеджеры не успевали отвечать в первые 5 минут — клиент уходил к конкуренту. В нерабочее время заявки копились без ответа.

## Решение

Разработали AI-бота на базе конечного автомата (FSM) с 8 состояниями и 22 сигналами распознавания. Бот подключён к Авито, Telegram и VK одновременно.

**Что делает бот:**
- Мгновенно отвечает на входящее сообщение (< 3 сек)
- Квалифицирует клиента: вид услуги, гео, срочность
- Обрабатывает возражения по цене и срокам
- Собирает контакт, адрес, удобное время
- Записывает на замер/консультацию
- Отправляет уведомление менеджеру с полной сводкой
- Защищён от prompt-injection и off-topic

**Архитектура FSM-промпта:**
```
greeting → qualify_what → qualify_geo → ask_contact → ask_address → ask_datetime → done → escalated
```

**22 сигнала классификации:** SERVICE_INFO, GEO_VALID, PRICE_Q, OBJECTION_PRICE, CONTACT, ADDR_FULL, DEMAND_HUMAN, REFUSE, ABUSE, INJECTION и др.

**Интеграции:**
- Авито, Telegram, VK — единый бот на все каналы
- CRM Битрикс24 — автосоздание сделки
- Telegram-уведомления менеджеру
- Google Calendar — запись на слот

## Результат

| Метрика | До | После |
|---------|-----|-------|
| Время первого ответа | 15-40 мин | < 3 сек |
| Конверсия заявка → замер | ~30% | 70%+ |
| Обработка в нерабочее время | 0% | 100% |
| Нагрузка на менеджеров | 100% | ~30% (только сложные кейсы) |

## Для кого подходит

Любая сервисная компания с входящим потоком из мессенджеров и досок объявлений: ремонт окон, сантехника, клининг, установка кондиционеров, ремонт техники, доставка.
